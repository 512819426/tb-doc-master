# Writing Style Guide

## Language

Use formal Chinese for user-facing proposal content unless the user requests another language.

Style goals:

- Mature
- Specific
- Restrained
- Verifiable
- Project-oriented
- Low in AI-like repetition

## Avoid

- Empty slogans
- Excessive "先进、领先、智能、高效、赋能"
- Unsupported "完全满足、行业第一、最佳、唯一"
- Repetitive "首先、其次、最后"
- Long generic background paragraphs
- Claims not backed by provided materials
- Over-promising service, security, delivery, or compliance commitments

## Prefer

- Requirements from the procurement document
- Concrete actions
- Deliverables
- Roles and responsibilities
- Milestones
- Control measures
- Acceptance support
- Evidence and materials

## Good proposal paragraph pattern

Use a natural structure:

1. State understanding of the requirement.
2. Describe the response measure.
3. Explain how it will be implemented.
4. Identify deliverables or evidence.
5. Add controls, review points, or guarantees if supported.

## Placeholder rules

Use visible placeholders for missing facts:

```text
【待补充：内容】
【待确认：内容】
【需根据采购文件格式调整：内容】
```

Collect all placeholders at the end.

## Defense/classified tone

For defense or classified projects:

- Be more conservative and compliance-focused.
- Avoid promotional language.
- Avoid claiming classified capability without evidence.
- Do not describe sensitive operational details beyond what is necessary for the bid.
- Mark all uncertain qualification, personnel, and environment claims for confirmation.
