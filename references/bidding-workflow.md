# Bidding Workflow

## Phase 1: File inventory

List all source files, identify file type, purpose, and whether each file has been read. Confirm whether the procurement file includes all appendices and forms.

Output:

- Received file list
- Missing critical file list
- Drafting assumptions

## Phase 2: Procurement document decomposition

Extract the following:

- Project name, project number, purchaser, agency, procurement method
- Budget, maximum price, package/lot structure, bid bond, service period, delivery location
- Deadline, opening time, submission method, electronic platform rules
- Bidder eligibility
- Consortium, subcontracting, branch company, manufacturer authorization rules
- Technical requirements
- Commercial requirements
- Contract terms
- Price requirements
- Bid document composition
- Signature, seal, copy, electronic file, encryption, and upload requirements
- Disqualification clauses
- Scoring method

Use tables for traceability. Include source clause numbers or section names whenever available.

## Phase 3: Risk-first review

Before drafting, classify risks:

- High: possible disqualification or invalid bid
- Medium: possible major score loss or serious clarification risk
- Low: affects completeness, presentation, or review impression

Do not bury risks inside narrative text. Present them in a table.

## Phase 4: Scoring strategy

Build a scoring matrix before writing chapters. Every high-value score item should map to:

- Proposal chapter
- Evidence material
- Drafting strategy
- Risk or missing proof

Write to score, not merely to describe.

## Phase 5: Proposal architecture

Generate the table of contents from the procurement document first. If the document has mandatory forms or order, follow it strictly. If not, use a conventional structure.

Separate qualification, commercial, technical, and price volumes when required.

## Phase 6: Draft generation

Draft in this order:

1. Qualification and fixed forms
2. Commercial response and commitments
3. Technical response and solution
4. Project implementation and management
5. Quality, security, confidentiality, training, and after-sales chapters
6. Deviation tables and response tables
7. Final checklist

Use placeholders for missing facts and collect them at the end.

## Phase 7: Final review

Check:

- All mandatory file components are present
- All eligibility items have evidence
- All scoring items have responses and proof
- Technical requirements are covered
- Commercial terms are covered
- Deviation tables match the narrative
- Placeholder list is complete
- Signature, seal, format, deadline, upload, and file naming requirements are covered

End every full proposal task with "待补充/待确认事项" and "提交前检查清单".
