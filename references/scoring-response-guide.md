# Scoring Response Guide

## Build the matrix first

When a scoring method exists, create:

| Scoring item | Score | Procurement requirement | Response strategy | Evidence | Proposal location | Risk |
|---|---:|---|---|---|---|---|

Use this matrix before drafting the main chapters.

## Response principles

- Prioritize high-score items.
- Map objective scores to documentary evidence.
- Map subjective scores to clear structure, implementation steps, deliverables, safeguards, and measurable outcomes.
- Use procurement wording where natural.
- Avoid broad praise and unsupported claims.
- Do not claim certificates, awards, cases, staff, platforms, patents, or manufacturer authorizations without user materials.

## Common scoring categories

### Company strength

Use only verified materials:

- Qualifications and certificates
- Management systems
- Financial or credit materials
- Awards and honors
- Similar project experience
- Product maturity evidence

### Technical solution

Respond with:

- Project understanding
- Overall architecture
- Functional response
- Technical route
- Integration and interface plan
- Data migration or governance plan
- Security design
- Deployment and operation plan
- Acceptance support

### Implementation capability

Respond with:

- Organization structure
- Role division
- Milestone plan
- Quality control
- Risk control
- Communication mechanism
- Change management
- Document deliverables

### Service and guarantee

Respond with:

- Service organization
- Response levels
- Maintenance process
- Training plan
- Emergency handling
- Knowledge transfer
- Continuous improvement

### Defense/classified items

Score responses must be cautious. Use only confirmed confidentiality qualification, personnel qualification, security management, classified carrier handling, and controlled delivery measures. Mark uncertain claims for confirmation.

## Output requirements

After drafting, verify that each scoring item has:

- A chapter response
- Evidence or placeholder
- No contradiction with deviation tables
- No unsupported claim
