# Proposal Structure

## Rule

Follow the procurement document's required structure and forms first. If the procurement document mandates a sequence, volume split, or template, use it exactly.

If no mandatory structure exists, use the conventional structure below and adjust to the project type.

## Conventional full proposal

1. Cover
2. Table of contents
3. Bid letter
4. Legal representative identity certificate
5. Authorization letter
6. Bid commitment letter
7. Bidder qualification documents
8. Company profile
9. Similar project experience
10. Project team
11. Commercial response
12. Technical response
13. Project understanding
14. Overall solution
15. Technical architecture
16. Functional response
17. System integration plan
18. Implementation plan
19. Project management plan
20. Quality assurance plan
21. Security and confidentiality plan
22. Training plan
23. After-sales and operation support plan
24. Emergency response plan
25. Commercial deviation table
26. Technical deviation table
27. Price documents
28. Other required attachments

## Software project additions

Consider:

- Requirements analysis
- System architecture
- Functional module design
- Data model or data governance
- Interface integration
- Deployment architecture
- Testing and acceptance
- Operation and maintenance
- Documentation deliverables

## System integration additions

Consider:

- Hardware/software list
- Network or environment plan
- Installation and commissioning
- Integration testing
- Cutover or migration
- Compatibility
- On-site implementation
- Spare parts or warranty

## Defense/classified additions

Consider:

- Confidentiality management system
- Classified personnel management
- Classified carrier management
- Site and environment security
- Data and document security
- Domestic substitution and supply chain control
- Security inspection and acceptance support
- Confidentiality commitment

## Volume split

If required, separate:

- Qualification volume
- Commercial volume
- Technical volume
- Price volume
- Electronic response files

Never merge price content into technical or commercial volumes if the procurement document forbids it.
