# Required Inputs

## Minimum files

Request these files before producing a complete proposal draft:

- Procurement, tender, RFP, RFQ, competitive consultation, competitive negotiation, or inquiry document
- All appendices, forms, annexes, technical specifications, drawings, or data sheets
- Clarification, Q&A, addendum, amendment, or correction files
- Bid response format files when separately provided
- Scoring method and evaluation criteria if not included in the main document

If any minimum file is missing, continue only for partial analysis and clearly mark the missing file impact.

## Company and project materials

Request these when drafting a complete proposal:

- Company profile and business license information
- Qualification certificates, security/confidentiality certificates, quality certificates, and validity dates
- Product, platform, solution, or system introduction
- Similar project cases with customer, contract scope, amount, date, and proof materials
- Project team names, roles, resumes, certificates, and social insurance/proof requirements if needed
- Delivery methodology, project management system, quality system, security system, confidentiality system
- After-sales service locations, response times, hotline, maintenance team, and spare parts capability if applicable
- Financial, tax, credit, litigation, or compliance materials if required by the procurement file
- Pricing table, quotation rules, bill of quantities, or pricing assumptions when price content is requested

Do not invent any of these items. Use placeholders and a missing-materials list.

## Template materials

Ask whether the user has:

- Word proposal template
- Required cover page, table of contents, headers, footers, fonts, margins, or numbering rules
- Customer-provided response forms
- Historical proposal for a similar project
- Company standard chapters or boilerplate

If no template exists, draft a formal structure based on the procurement document and conventional bid practice.

## Completeness output

At the start of a task, output:

| Category | Received | Missing | Impact |
|---|---|---|---|

Use "Impact" to explain whether the work can proceed as complete drafting, partial drafting, or only analysis.
